## Quick start

**Requirements:** Python **3.10+**, a **[Google AI Studio API key](https://aistudio.google.com/app/apikey)** for Gemini.

**1.** Clone the repo and enter the project root.

**2.** Create an environment and install the package:

```bash
# Option A — Conda
conda env create -f environment.yml
conda activate prob-desk

# Option B — venv
python3 -m venv .venv && source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -e .
```

**3.** Configure env and run:

```bash
cp .env.example .env
# Edit .env: set GOOGLE_API_KEY (required). Leave GOOGLE_GENAI_USE_VERTEXAI=false.
prob-desk
```

At the prompt, try something like: *Summarize open KXHIGHNY Kalshi markets and risks.*

## ADK Web (browser UI)

Use the official **ADK Web** chat UI to talk to your agents in the browser (sessions, event history, state). ADK Web is **for local development only**, not production — see [Web interface](https://google.github.io/adk-docs/runtime/web-interface/).

### Install the ADK CLI

The **`adk`** command ships with **`google-adk`**, which is already a dependency of this project. After `pip install -e .`, you should have `adk` on your `PATH`. If not:

```bash
pip install google-adk
adk --help
```

### Run the server from the repo root

`adk web` takes an optional **agents directory**; if you omit it, the CLI uses your **current working directory** (same as passing `.`). So from the **repository root**, after [Quick start](#quick-start) (`pip install -e .`, `.env` with **`GOOGLE_API_KEY`**):

```bash
cd /path/to/ProbDesk
adk web
```

Optional: `adk web . --port 8000 --host 127.0.0.1` (defaults are port **8000** and host **127.0.0.1** — see [ADK Web docs](https://google.github.io/adk-docs/runtime/web-interface/)).

### Open the UI

In the browser, open **http://127.0.0.1:8000** (or your chosen port). Pick the agent exposed under the [layout ADK expects](https://google.github.io/adk-docs/runtime/web-interface/) (subfolders with `agent.py` / `root_agent`). This project’s graph is defined under [`prob_desk/agents/`](prob_desk/agents/); `root_agent` is exported from [`prob_desk.agents`](prob_desk/agents/__init__.py).

## Optional: Kalshi account

Public market data works against the **demo** API without keys. For **balance, positions, orders**, set in `.env`:

- `KALSHI_API_KEY_ID`
- `KALSHI_PRIVATE_KEY_PATH` (path to your `.pem`) **or** `KALSHI_PRIVATE_KEY_PEM`
- `KALSHI_TRADE_API_BASE` — must match your key (**demo** vs **production**)

Then you can ask things like *summarize my portfolio* (Kalshi only).

## Tactical policy (RL scripts)

Train / evaluate the synthetic execution simulator **without** the LLM:

```bash
python scripts/execution/train_tactical_policy.py --episodes 400 --out-dir models
python scripts/execution/eval_execution.py --synthetic --episodes 100 --out-dir outputs/kalshi_eval
```

Details: [`docs/tactical-policy-cli-scripts.md`](docs/tactical-policy-cli-scripts.md). Example training YAML: [`prob_desk/execution/tactical_policy.example.yaml`](prob_desk/execution/tactical_policy.example.yaml).
